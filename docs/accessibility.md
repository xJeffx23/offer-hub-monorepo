## 1. Commitment to Accessibility

OFFER-HUB ("the Platform") is committed to ensuring digital accessibility for all users, including those with disabilities. We continuously work to improve the accessibility and usability of the Platform to conform to the **Web Content Accessibility Guidelines (WCAG) 2.1 Level AA** standard and applicable accessibility laws, including the EU Web Accessibility Directive and Section 508 of the Rehabilitation Act.

---

## 2. Conformance Status

### Target Conformance Level

We aim to achieve and maintain **WCAG 2.1 Level AA** compliance across the Platform. This standard covers accessibility for individuals with:

- Vision impairments (blindness, low vision, color blindness)
- Hearing impairments (deafness, hard of hearing)
- Motor impairments (reduced mobility, tremors)
- Cognitive impairments (dyslexia, learning disabilities)
- Vestibular disorders and seizure disorders

WCAG 2.1 Level A is the foundational compliance level, while Level AA represents the industry standard target. Level AAA is considered aspirational and is not a required standard for most organizations.

### Scope of Compliance

Our accessibility efforts cover:

- ✅ The OFFER-HUB website and web application (https://offer-hub.tech)
- ✅ All public-facing pages and features
- ✅ User account and dashboard interfaces
- ✅ Documentation and help resources
- ✅ API documentation and code examples

### Last Accessibility Review

- **Date of Last Full Audit:** April 2026
- **Next Scheduled Review:** October 2026
- **Review Methodology:** Combination of automated accessibility scanning tools and manual testing by accessibility specialists

## 3. Known Accessibility Limitations

While we strive for full compliance, the following known limitations exist and are planned for remediation:

### Current Limitations

1. **PDF Documents**
   - Some older PDF documents may not have proper accessibility tags or alternative text
   - **Remediation Timeline:** Q3 2026 — all PDFs will be converted to accessible formats or replaced with web-native content

2. **Third-Party Embedded Components**
   - Certain third-party integrations (e.g., payment processors, analytics) may have limited accessibility support
   - **Remediation Plan:** We work with vendors to enforce accessibility requirements in service agreements

3. **Real-Time Data Visualizations**
   - Interactive charts and real-time data displays on the dashboard may require keyboard navigation enhancements
   - **Remediation Timeline:** Q2 2026 — enhanced keyboard support and data table alternatives will be provided

4. **Video Content**
   - Some archived video content may lack captions or transcripts
   - **Remediation Timeline:** Q3 2026 — all new videos will include captions; existing videos will be progressively updated

5. **Blockchain Transaction Details**
   - Complex blockchain transaction data may be difficult to present in fully accessible ways while maintaining accuracy
   - **Ongoing Effort:** We provide alternative text representations and detailed explanations for all transaction states

### Technical Accessibility Features

✅ **Implemented Features:**
- Semantic HTML structure with proper heading hierarchy
- ARIA labels and roles for dynamic content
- Keyboard navigation support (Tab, Enter, Escape)
- High contrast mode support
- Focus indicators for keyboard navigation
- Alternative text for images and icons
- Language declarations for screen readers
- Skip-to-content links
- Responsive design supporting text resizing
- Support for browser zoom and system font size settings
- Color not used as sole means of conveying information

## 4. How to Report Accessibility Issues

We value your feedback and are committed to making the Platform accessible to everyone. If you encounter any accessibility barriers while using OFFER-HUB, please report them to us:

### Primary Contact Method

**Email:** [support@offerhub.io](mailto:support@offerhub.io)

**Subject Line:** Accessibility Issue Report

**Information to Include:**
- Description of the accessibility barrier
- The page or feature where you encountered the issue
- Your browser and assistive technology (if applicable)
- The specific WCAG criterion that appears to be violated (if known)
- Steps to reproduce the issue

### Alternative Contact Methods

For users who cannot use email or the standard contact form:

1. **Telegram:** [@offer_hub_contributors](https://t.me/offer_hub_contributors)
   - Message the support team about accessibility concerns

2. **Discord:** [OFFER-HUB Community Server](https://discord.gg/yH4vBNWwc)
   - Post in the #support channel with accessibility questions

3. **GitHub Issues:** [OFFER-HUB GitHub Repository](https://github.com/OFFER-HUB)
   - Create an issue labeled with `accessibility` tag for technical problems

### Accessibility Request Process

When you report an accessibility issue, we commit to:

1. **Acknowledgment:** Respond within 2 business days to confirm receipt of your report
2. **Assessment:** Evaluate the issue and determine severity
3. **Remediation:** Provide a timeline for fixing the issue or explain why it cannot be resolved
4. **Follow-up:** Notify you when the issue has been resolved

## 6. Legal Compliance

### Applicable Laws and Standards

OFFER-HUB complies with or works toward compliance with:

- **Web Content Accessibility Guidelines (WCAG) 2.1 Level AA** — International standard
- **EU Web Accessibility Directive (2016/2102)** — Required for public sector bodies and certain private entities in EU
- **Section 508 of the Rehabilitation Act** — Required for US federal agencies and contractors
- **ADA (Americans with Disabilities Act)** — Applies to certain businesses in the United States
- **AODA (Accessibility for Ontarians with Disabilities Act)** — Applies to organizations in Ontario, Canada

### Disproportionality Assessment

While we strive for full compliance, certain accessibility measures may be determined as "disproportionate" under applicable laws. Any disproportionality determinations are made based on:

- Technical feasibility
- Cost-benefit analysis
- Impact on core functionality
- Industry best practices
